# Expense Tracker — учёт личных финансов

## Описание проекта

**Expense Tracker** — это веб‑приложение на Django для учёта личных финансов. Пользовательские интерфейсы реализованы с помощью HTML‑шаблонов Django. Проект помогает пользователю контролировать доходы и расходы, распределять их по категориям и счетам, устанавливать бюджеты и анализировать финансовое поведение с помощью отчётов и графиков.
В проекте реализована встроенная аутентификация, поэтому каждый пользователь работает только со своими данными.

________________________________________

Идея проекта

Помочь пользователю: - записывать транзакции (доходы и расходы); - группировать их по категориям и счетам; - устанавливать бюджеты по категориям; - получать сводные отчёты и аналитику; - понимать, на что уходит больше всего денег и укладывается ли он в бюджет.

________________________________________

Используемые технологии

• Python 3.10+
• Django 4+
• SQLite (по умолчанию)
• Bootstrap 5 (интерфейс)
• Chart.js (диаграммы)

________________________________________

Модели ORM

User
Назначение: - владелец данных; - авторизация и аутентификация.

________________________________________

Account (Счёт)
Назначение: - хранение информации о финансовых счетах пользователя.
Функционал: - текущий баланс; - автоматический пересчёт баланса при создании / изменении / удалении транзакций; - фильтрация транзакций по счёту; - используется в отчётах («Расходы по счёту»).

________________________________________

Category (Категория)
Назначение: - классификация доходов и расходов.
Функционал: - создание пользовательских категорий; - иерархия категорий (например: Еда → Продукты); - используется в аналитике и бюджетах.

________________________________________

Transaction (Транзакция)
Назначение: - основная финансовая запись.
Функционал: - создание доходов и расходов; - изменение баланса связанного счёта; - привязка к категории и счёту; - используется в отчётах и аналитике.

________________________________________

Budget (Бюджет)
Назначение: - контроль расходов по категориям.
Функционал: - установка лимита расходов за период (обычно месяц); - расчёт: - потраченной суммы; - оставшейся суммы; - превышения бюджета.

________________________________________

Основной функционал проекта

Учёт транзакций
Пользователь может: - добавлять доходы и расходы; - указывать сумму, дату и описание; - выбирать категорию и счёт.

________________________________________

Категории
• создание собственных категорий;
• поддержка иерархии категорий;
• используются для анализа и отчётов.

________________________________________

Счета (Accounts)
• создание нескольких счетов (наличные, карта, кошелёк);
• отображение текущего баланса;
• автоматическое обновление баланса.

________________________________________

Бюджеты
• установка месячного бюджета по категориям;
• отображение:
o сколько потрачено;
o сколько осталось;
o превышен ли лимит.

________________________________________

Список транзакций с фильтрами
Поддерживаются фильтры: - по дате (от / до); - по категории; - по счёту; - по типу (доход / расход).

________________________________________

Отчёты и аналитика
В проекте предусмотрен блок аналитики: - сводка доходов и расходов за месяц; - разбивка по категориям; - диаграммы (Pie chart, Bar chart);
Пользователь может понять: - на что уходит больше всего денег; - как меняются траты по месяцам; - укладывается ли он в бюджет.

________________________________________

Административная панель
Админка Django позволяет: - управлять всеми моделями; - просматривать и фильтровать данные; - видеть транзакции внутри карточки счёта; - управлять пользователями.

________________________________________

Авторизация и приватность данных
• регистрация;
• вход / выход;
• каждый пользователь видит только свои данные.

Алгоритм эксплуатации приложения “Expense Tracker”

Шаг 1: Регистрация или вход
• Перейдите на главную страницу приложения.
• Если у вас ещё нет аккаунта:
o Нажмите “Регистрация”.
o Заполните форму: имя пользователя, email, пароль.
o Подтвердите регистрацию.
• Если аккаунт уже есть:
o Нажмите “Войти”.
o Введите имя пользователя и пароль.

________________________________________

Шаг 2: Настройка счетов
• После входа перейдите в раздел “Счета”.
• Создайте минимум один счёт (например: “Наличные”, “Карта”, “Счёт в банке”).
• Укажите начальный баланс, если он есть.
• Баланс будет автоматически обновляться при добавлении транзакций.

________________________________________

Шаг 3: Настройка категорий
• Перейдите в раздел “Категории”.
• Создайте категории для доходов и расходов (например: “Еда”, “Транспорт”, “Зарплата”).
• При необходимости можно создать вложенные категории (например: “Еда → Продукты”).

________________________________________

Шаг 4: Добавление транзакций
• Перейдите в раздел “Добавить транзакцию”.
• Выберите тип: доход или расход.
• Заполните:
o Сумма
o Дата
o Описание (необязательно)
o Счёт
o Категория
• Нажмите “Сохранить”. Баланс счёта обновится автоматически.

________________________________________

Шаг 5: Работа с бюджетами (опционально)
• Перейдите в раздел “Бюджеты”.
• Выберите категорию расходов.
• Установите месячный лимит.
• В разделе “Бюджеты” вы увидите:
o Потрачено в текущем месяце
o Остаток
o Превышение лимита (если есть)

________________________________________

Шаг 6: Просмотр транзакций
• Перейдите в раздел “Транзакции”.
• Используйте фильтры:
o По дате (от/до)
o По категории
o По счёту
o По типу (доход/расход)

________________________________________

Шаг 7: Аналитика и отчёты
• Перейдите в раздел “Аналитика” или “Отчёты”.
• Здесь вы увидите:
o Сводку доходов и расходов за месяц
o Распределение трат по категориям (в виде диаграмм)
o Сравнение с бюджетами

________________________________________

Шаг 8: Редактирование и удаление данных
• В разделах “Счета”, “Категории”, “Транзакции” вы можете:
o Изменить существующие записи (нажать на запись → “Изменить”)
o Удалить ненужные данные (кнопка “Удалить”)

________________________________________

Шаг 9: Выход из системы
• Нажмите “Выйти” в правом верхнем углу, чтобы завершить сеанс.

________________________________________

Дополнительно: Административная панель
• Если вы администратор:
o Перейдите по адресу /admin/.
o Войдите под учётной записью суперпользователя.
o Здесь вы можете управлять всеми моделями и пользователями.
